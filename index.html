2. Security Implementation Strategies

1 Middleware Query Interceptor

Intercept all queries and automatically inject affiliate filter

# In MCP Server (Python) def secure_query(user_id, base_query): affiliate_id = get_user_affiliate(user_id) # Parse and inject WHERE clause if "WHERE" in base_query.upper(): secured = base_query.replace( "WHERE", f"WHERE affiliate_id = '{affiliate_id}' AND" ) else: secured = base_query + f" WHERE affiliate_id = '{affiliate_id}'" return secured

2 Row-Level Security (RLS) in SQL Server

Use SQL Server's built-in RLS to enforce security at database level

-- Create security policy CREATE FUNCTION dbo.fn_securitypredicate(@affiliate_id AS VARCHAR(50)) RETURNS TABLE WITH SCHEMABINDING AS RETURN SELECT 1 AS fn_securitypredicate_result WHERE @affiliate_id = CAST(SESSION_CONTEXT(N'affiliate_id') AS VARCHAR(50)); -- Apply to tables CREATE SECURITY POLICY AffiliateSecurityPolicy ADD FILTER PREDICATE dbo.fn_securitypredicate(affiliate_id) ON dbo.Opportunities, ADD FILTER PREDICATE dbo.fn_securitypredicate(affiliate_id) ON dbo.Accounts, ADD FILTER PREDICATE dbo.fn_securitypredicate(affiliate_id) ON dbo.Contacts; -- Set context before query EXEC sp_set_session_context @key = N'affiliate_id', @value = 'A';
‚úÖ Pros:
‚Ä¢ Database-enforced (most secure)
‚Ä¢ No query modification needed
‚Ä¢ Handles all query types automatically
‚ùå Cons:
‚Ä¢ Requires SQL Server Enterprise (or 2016+)
‚Ä¢ Session context must be set per connection

3 Parameterized Views/Stored Procedures

Create affiliate-specific database objects that users access through MCP

-- Create parameterized stored procedure CREATE PROCEDURE usp_GetOpportunityStatus @affiliate_id VARCHAR(50), @user_filter NVARCHAR(MAX) = NULL AS BEGIN SELECT o.opportunity_id, o.status, a.account_name, c.contact_name FROM Opportunities o JOIN Accounts a ON o.account_id = a.account_id JOIN Contacts c ON o.contact_id = c.contact_id WHERE o.affiliate_id = @affiliate_id AND (@user_filter IS NULL OR o.status LIKE '%' + @user_filter + '%') END -- MCP calls with affiliate parameter EXEC usp_GetOpportunityStatus @affiliate_id = 'A', @user_filter = 'Open'
‚úÖ Pros:
‚Ä¢ Simple and reliable
‚Ä¢ Works on any SQL Server version
‚Ä¢ Clear audit trail
‚ùå Cons:
‚Ä¢ Less flexible for ad-hoc queries
‚Ä¢ Requires procedure for each query type
